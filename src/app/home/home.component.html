Data Attribute Configuration

Use HTML data attributes on the script tag for configuration. The analytics auto-initializes when the page loads.

Attribute Type Default Description
data-api-key string (required) Required: Your unique website/app identifier for reports.
data-batch-interval number 30000 How often to send batched events (milliseconds).
data-debug boolean false Enable console logging for debugging.
data-disable-scroll boolean false Disable automatic scroll depth tracking.
data-disable-clicks boolean false Disable automatic click tracking.
data-endpoint string https://api.simpletrack.dev Custom API endpoint for event submission.

  • Network: Check for failed API requests to our endpoint
  • CORS: Ensure your domain allows requests to api.simpletrack.dev

How do I get my analytics reports?
Contact us with your serviceName to access your website's analytics data and reports.
Is it free? Open-source?
SDK is free and open-source (MIT licensed). Hosting and report access may have associated costs.
What data is collected?
Only what you track: anonymized UUIDs, browser/session info, events. No IP logging by default.
Works on React/Vue/etc.?
Yes! Use window.STKAnalytics in components - no wrapper code needed. See component examples above.
Minification?
Download includes minified version. Use a bundler like Webpack for production.
Can I host the data myself?
Currently, all data is sent to our central endpoint. Custom hosting may be available in enterprise plans.
How do I track custom events?
Use window.STKAnalytics.trackEvent('event_name', { data }) anywhere in your code.
What's the difference between debug and production?
Debug mode logs to console but doesn't send data. Production sends data but no console logs.
