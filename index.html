SimpleTrack is a lightweight, zero-configuration web analytics SDK designed for modern web applications. With just 5KB of JavaScript, you get automatic page tracking, custom event monitoring, user analytics, performance metrics, and comprehensive reporting capabilities. Perfect for developers who want powerful analytics without the complexity of traditional tracking solutions.

Step 1: Include the Script

Add this to the <head> or <body> of your HTML:

<script src="https://simpletrack.dev/stk-analytics.min.js"></script>

Step 2: Initialize

After the script loads, initialize with your unique service name. This identifies your website in the analytics reports.

<script>
  STKAnalytics.init({
    serviceName: 'my-website-name'
  });
</script>

Note: Each website should use a unique serviceName to separate their analytics data and reports. Contact us to get your reports.

Using in Angular/React/Vue Components

The script automatically creates a global STKAnalytics object for easy component access:

// In any component - no wrapper code needed!
window.STKAnalytics.trackEvent('button_click', { button: 'signup' });
window.STKAnalytics.trackPromoImpression('promo_123');
window.STKAnalytics.trackPromoClick('promo_123', 'learn_more');
window.STKAnalytics.trackError('API Error', { code: 500 });

Using in Regular JavaScript Apps

For vanilla JavaScript applications without frameworks, use the same global analytics object:

// After script loads and initializes
document.getElementById('signup-btn').addEventListener('click', function() {
  window.STKAnalytics.trackEvent('signup_click', { 
    source: 'hero_section',
    plan: 'free'
  });
});

// Track form submissions
document.getElementById('contact-form').addEventListener('submit', function(e) {
  window.STKAnalytics.trackEvent('form_submit', {
    form: 'contact',
    category: 'engagement'
  });
});

// Track custom interactions
function trackCustomAction(action, data) {
  window.STKAnalytics.trackEvent(action, data);
}

Configuration Options

Pass an object to STKAnalytics.init(options). The apiUrl is automatically configured to our central analytics endpoint.

Option Type Default Description
serviceName string 'unknown' Required: Your unique website/app identifier for reports.
apiUrl string Production URL Override for testing (e.g., localhost). Defaults to our production endpoint.
batchIntervalMs number 5000 Batch send interval in ms (e.g., 10000 for 10s).
debug boolean false Enable console logs for troubleshooting. Note: In debug mode, data is NOT sent to backend.
autoTrackPageViews boolean true Auto-track page views on load.
autoTrackClicks boolean true Auto-track clicks on [data-click] elements.
autoTrackScroll boolean true Track max scroll depth (%).
autoTrackPerformance boolean true Track page load times.

Example with Custom Config:

STKAnalytics.init({
  serviceName: 'my-ecommerce-site',
  batchIntervalMs: 10000,
  debug: true,
  autoTrackScroll: false  // Disable if not needed
});

Testing with Local Backend:

STKAnalytics.init({
  serviceName: 'my-website-name',
  apiUrl: 'http://localhost:3004/analytics/log',  // Your local endpoint
  debug: true  // Enable debug to see logs
});

Update config later with STKAnalytics.updateConfig({ debug: true }).

Implementation Steps:

  1. Initialize analytics (automatic tracking starts)
  2. Add data-click attributes to elements you want to track
  3. SPA users: call trackNavigation() on route changes
  4. Review automatic data collection

Once initialized, STK Analytics tracks common interactions out-of-the-box. No extra code needed!

  • Page Views: Fires page_view on load, including attribution (UTM params, referrer).
  • Session Start: Logs session_start with visit count and first-visit timestamp.
  • Scroll Depth: Tracks max scroll % (e.g., 25%, 50%) via scroll events.
  • Clicks: Listens for clicks on elements with data-click="button|link|etc". Example: <button data-click="purchase">Buy Now</button> ‚Üí click event with label and coords.
  • Page Exit/Hidden: Captures page_exit or page_hidden with time-on-page and scroll data.
  • Performance: Logs page_load_complete with load time, DOM ready, etc.

All events include: user ID (UUID), session ID, browser info (UA, screen res, timezone), attribution, and page path.

Adding Click Tracking

Add data-click attributes to elements for automatic click tracking:

<!-- Examples -->
<button data-click="signup">Sign Up</button>
<a href="/pricing" data-click="pricing_link">View Pricing</a>
<div data-click="promo_banner" class="promo">Special Offer!</div>

SPA Navigation Tracking

For Single Page Applications, track route changes:

// Angular example
import { Router, NavigationEnd } from '@angular/router';

constructor(private router: Router) {
  this.router.events.subscribe(event => {
    if (event instanceof NavigationEnd) {
      window.STKAnalytics.trackNavigation(event.url, {
        previousUrl: this.router.url
      });
    }
  });
}

Implementation Steps:

  1. Use trackEvent() for custom business events
  2. Leverage convenience methods for common patterns
  3. Call sendBatch() for immediate sending on important events
  4. Structure event data consistently

Use STKAnalytics.trackEvent(name, data) for custom logic. data can include category, label, value, and custom object.

Basic Event

STKAnalytics.trackEvent('signup_complete', {
  category: 'conversion',
  label: 'email-signup',
  value: 1,
  custom: { plan: 'premium' }
});

Convenience Methods

  • Interactions: STKAnalytics.trackInteraction('button_click', 'Add to Cart')
  • Navigation: STKAnalytics.trackNavigation('/checkout', { step: 'payment' })
  • Errors: STKAnalytics.trackError('Payment failed', { errorCode: 500 })
  • Performance: STKAnalytics.trackPerformance('api_call', 250, { endpoint: '/users' }) (ms)

Force Send Batch

Send queued events immediately: await STKAnalytics.sendBatch(); (useful on form submits).

// Example: Send immediately on form submit
document.getElementById('contact-form').onsubmit = async function() {
  window.STKAnalytics.trackEvent('form_submit', { form: 'contact' });
  await STKAnalytics.sendBatch(); // Send immediately
};

Implementation Steps:

  1. Call setUserEmail() after successful login
  2. Call clearUserEmail() on logout
  3. All subsequent events will be tagged as authenticated
  4. User email is stored in localStorage for persistence

Track authenticated users by setting email (stored in localStorage for persistence).

// After login
STKAnalytics.setUserEmail('user@example.com');

// Clear on logout
STKAnalytics.clearUserEmail();

Events will include user_email and user_type: 'authenticated'. Anonymous users default to UUID.

Integration Examples

// React login component
const handleLogin = async (email, password) => {
  const response = await loginAPI(email, password);
  if (response.success) {
    window.STKAnalytics.setUserEmail(email);
    // Redirect to dashboard
  }
};

// Angular auth service
@Injectable( { providedIn: 'root' })
export class AuthService {
  login(credentials: any) {
    return this.http.post('/api/login', credentials).pipe(
      tap(response => {
        if (response.token) {
          window.STKAnalytics.setUserEmail(credentials.email);
          localStorage.setItem('token', response.token);
        }
      })
    );
  }

  logout() {
    window.STKAnalytics.clearUserEmail();
    localStorage.removeItem('token');
    // Redirect to login
  }
}

Privacy Considerations

  • Email is stored locally (localStorage) - never sent to our servers
  • Only used to tag events as authenticated vs anonymous
  • Clear on logout to maintain privacy
  • Complies with basic GDPR requirements

Implementation Examples

// Banner component with intersection observer
class PromoBanner extends Component {
  componentDidMount() {
    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          window.STKAnalytics.trackPromoImpression('summer_sale_2024', {
            position: 'sidebar',
            bannerSize: '300x250'
          });
          observer.disconnect(); // Track only once
        }
      });
    });
    observer.observe(this.bannerRef.current);
  }

  handleClick = () => {
    window.STKAnalytics.trackPromoClick('summer_sale_2024', 'shop_now');
    // Navigate to product page
  };

  render() {
    return (
      <div ref={this.bannerRef} onClick={this.handleClick}>
        Summer Sale - 50% Off!
      </div>
    );
  }
}

Troubleshooting Guide

‚ùå No events sending?

  • Check browser console for errors
  • Verify serviceName is set
  • Check network tab for POST requests to our endpoint
  • Ensure debug mode is disabled for production

‚ùå Storage issues?

  • Falls back to in-memory UUIDs if localStorage fails
  • Check browser privacy settings
  • Some browsers block third-party storage

‚ùå SPA navigation not tracking?

  • Call trackNavigation() on route changes
  • See Automatic Tracking section for examples

Privacy & Security

Privacy Notes: Uses sessionStorage for sessions, localStorage for user ID/email (opt-in). No third-party trackers. Complies with basic GDPR (no PII without consent).

  • No cookies by default (respects privacy)
  • Anonymous UUIDs for non-authenticated users
  • Optional email storage for authenticated tracking
  • All data sent over HTTPS to our secure endpoint
